# Plan 15.2: NLP Direct Link Extraction & Layout Alignment

## Objective
Enable multiple direct link parsing within tasks and subtasks, rendering them as hostname badges inline. Automatically extract URLs from the task text during registration/editing to create subtasks for each, and remove them from the main task title (removing the need for separate URL input fields). Standardize all task registration forms (Daily Log, DayTasksModal, FutureLog, MonthlyLog, SomedayMaybe) to have equivalent metadata capabilities, keeping only the "captura rápida" simplified for mobile display. Fix text wrapping and icon alignment issues in the Daily Log BulletItem layout.

## Tasks
- [x] **Task 1: NLP Link Detection and Subtask Generation**
  - Implement link parsing helper `extractLinksFromText` in `src/utils/plannerUtils.ts`.
  - Update `useBujoItems.ts` save standard, someday, and edit handlers to extract links, strip them from the task title, and generate subtasks for each link automatically.
  - Update `BujoContext.tsx` rapid capture and suggestions creation handlers to also parse text links.
- [x] **Task 2: UI Removals & Standardization**
  - Remove all separate URL input fields, toggle buttons, and associated states across all log creation tabs (`DailyLogTab.tsx`, `DayTasksModal.tsx`, `FutureLogTab.tsx`, `MonthlyLogTab.tsx`, `SomedayMaybeTab.tsx`).
  - Standardize `FutureLogTab.tsx` and `SomedayMaybeTab.tsx` creation forms to support type selector, time/hour input, icon selection, energy, complexity, and executionTime fields.
  - Keep `RapidLog.tsx` (captura rápida) simplified and optimized for mobile screens.
- [x] **Task 3: Daily Log Layout Bug Fix**
  - Refactor `BulletItem.tsx` layout to use flex-row and top alignment (self-start) on all screens instead of stacking (flex-col) on mobile.
  - Set text wrapper container to flex-1 with min-w-0 and break-words to guarantee correct wrapping, keeping action icons (AI, subtask, edit, delete) pinned side-by-side on the right.
- [x] **Task 4: Production Build Compilation Check**
  - Verify that `npm run build` runs and exits with code 0.

## Verification Criteria
1. Typing task names containing links (e.g. "Revisar docs google.com y github.com") extracts both URLs, removes them from the task text, and automatically creates two subtasks.
2. Direct links on both tasks and subtasks are rendered inline as clean hostname badges (e.g., `🔗 google.com`), hiding the full raw URL text.
3. No task creation/edit forms show a separate text input field for URLs.
4. FutureLog and SomedayMaybe creation forms include standard fields (hour, icon, energy, complexity, executionTime) matching the daily/monthly log forms, while the rapid capture modal remains simplified.
5. In the Daily Log, long tasks wrap correctly and their actions (AI, subtasks, edit, delete) remain visible and correctly aligned.
6. The project builds cleanly with no TypeScript or bundler errors.
