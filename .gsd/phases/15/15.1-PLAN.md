# Plan 15.1: Migration Duplication Fix & Unified Delay Tracking

## Objective
Fix the task duplication issue that occurs when migrating uncompleted/scheduled tasks to the next day. Ensure that instead of cloning the task, it is moved directly. Unify delay calculations and card aging formatting across all views (Daily Log, Someday/Maybe Tab, Weekly Log, etc.) using days and hours (no minutes or seconds).

## Tasks
- [x] **Task 1: Prevent Duplication in Migration Logic**
  - Update `migrateUncompletedTasksToNextDay` and `checkAutoMigration` in `src/hooks/useBujoItems.ts` to move tasks in-place by updating `date` instead of duplicating them.
- [x] **Task 2: Fix TypeScript Compiler Errors**
  - Resolve TS2367 errors in `src/hooks/useBujoItems.ts` caused by checking `item.status === 'migrated'` within code paths where `'migrated'` was already excluded.
- [x] **Task 3: Unify Task Delay & Card Aging Displays**
  - Implement unified `getTaskDelayInfo` helper in `src/utils/plannerUtils.ts`.
  - Update `src/features/planner/components/BulletItem.tsx` to render delay badge and age the cards based on total delay hours/days.
  - Update `src/features/planner/components/SomedayMaybeTab.tsx` (both sticky notes and lists) to use the same delay calculation and display.
  - Update `src/features/planner/components/WeeklyLogTab.tsx` preview tasks to show delay display using the new format.
- [ ] **Task 4: Run Production Build and Verify**
  - Execute production build using `npm run build` to confirm zero TS or bundler errors.

## Verification Criteria
1. Tasks manual/auto migration shifts the task date in-place; no duplicate is left on the previous day.
2. Delay strings only show days and hours (e.g. `1d 4h` or `12h` or `5d`), never minutes or seconds.
3. Card aging styles (`aged-paper-1/2/3`) apply to all tasks based on hour delay (if scheduled) or days since creation (if not scheduled).
4. No compilation/TypeScript errors occur during production build.
