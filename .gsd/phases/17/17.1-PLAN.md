# Plan 17.1: Close Modals on Escape Key Press

## Objective
Enable a consistent keyboard user experience by ensuring that pressing the `Escape` key closes any currently open modals or popovers across the application.

## Tasks
- [ ] **Task 1: Add Escape Key Listener to Common Modals**
  - Add listener to `ConfirmationModal.tsx`.
  - Add listener to `FeatureHelpModal.tsx`.
  - Add listener to `RapidLog.tsx`.
- [ ] **Task 2: Add Escape Key Listener to ADHD & Collections Modals**
  - Add listener to `EnergyRhythmModal.tsx`.
  - Add listener to `CollectionItemModal.tsx`.
- [ ] **Task 3: Add Escape Key Listener to Planner Modals**
  - Add listener to `AISuggestionsModal.tsx`.
  - Add listener to `DayTasksModal.tsx`.
  - Add listener to `GlobalSearchModal.tsx`.
  - Add listener to `OverloadReliefModal.tsx`.
- [ ] **Task 4: Production Build & Validation**
  - Ensure the project builds cleanly.

## Verification Criteria
1. Opening any modal (e.g., Quick Log via Space/Ctrl+K, Day Detail Modal, Search Modal, Help Modal, etc.) and pressing the `Escape` key immediately closes it.
2. If multiple nested overlays are open, the active modal closes correctly.
3. The project builds cleanly with no TypeScript/Vite errors.
