# Plan 11.1: Brag Document Integration, Sync & UI Layout

## Objective
Refine the docs/example-bradocs.md template, implement a Brag Document feature that feeds into the growth chart, ensure data syncs with Supabase, and reorganize the IndexTab layout.

## Context
The user wants to easily add daily Brag updates that visibly contribute to their professional/academic evolution chart. This data must persist to the cloud (Supabase) in addition to localStorage. Finally, the Index layout needs rearranging: Quick Access Menu at the top, and Knowledge Evolution Chart immediately below the Focus Center.

## Tasks

- [x] **Task 1: Refine docs/example-bradocs.md**
  - Type: auto
  - Description: Rewrite the document to serve as a high-quality template based on industry best practices (Julia Evans, Elton Minetto). Include sections for Goals, Major Projects & Impacts, Collaboration & Mentorship, Design & Architecture, Company Growth, and Continuous Learning.
  - Files: docs/example-bradocs.md

- [x] **Task 2: Implement Brag Document State, Types & Supabase Sync**
  - Type: auto
  - Description: Add BragEntry interface to types. Update BujoContext and Supabase sync logic to handle the new brag_entries state, ensuring it saves locally and syncs to the cloud just like items or collections.
  - Files: src/types/index.ts, src/context/BujoContext.tsx

- [x] **Task 3: Integrate Brag Document UI & Chart Logic**
  - Type: auto
  - Description: Add a Conquistas (Brag) sub-tab in KnowledgeEvolutionChart.tsx. Create a list view and a modal for adding daily Brag Entries (Title, Category, Scope, Impact, OKRs). Link adding a Brag Entry to the existing XP/Progress logic so that adding a Brag entry visibly bumps the chart/progress for the associated subject/category.
  - Files: src/features/education/components/KnowledgeEvolutionChart.tsx

- [x] **Task 4: Reorganize IndexTab Layout**
  - Type: auto
  - Description: Move the Menu de Acesso Rápido to the very top. Move the Knowledge Evolution Chart to be immediately below the Central de Foco (Welcome Banner/KPIs).
  - Files: src/features/planner/components/IndexTab.tsx

## Verification Criteria
1. docs/example-bradocs.md is updated.
2. IndexTab layout matches the new requested order.
3. Brag Document UI is present and allows adding daily entries.
4. Adding a Brag Entry updates the Knowledge Evolution chart metrics.
5. Brag entries persist across reloads and sync to Supabase (if connected).
