# Plan 10.1: Tutorial Mobile Responsiveness & UX Polish

## Objective
Ensure the onboarding tutorial is fully functional and visually correct on mobile devices, preventing balloon overflow and handling hidden UI elements.

## Context
The user reported that the tutorial balloon overflows on mobile, preventing access to "Next" buttons, and that feature highlights (spotlights) for the sidebar don't work because the sidebar is hidden on small screens.

## Tasks
- [ ] **Task 1: Adjust TutorialOverlay.tsx for Mobile**
  - Type: auto
  - Description: Add max-height and scrolling to the tutorial balloon. Redirect sidebar highlights to the mobile menu trigger.
  - Files: `src/components/common/TutorialOverlay.tsx`, `src/features/planner/components/Header.tsx`
  - Done: Balloon has `max-h-[85vh]` and `overflow-y-auto`. `id="tutorial-mobile-menu-trigger"` added to Header. Lógica de fallback para mobile implementada.

- [ ] **Task 2: Verification and Build**
  - Type: auto
  - Description: Verify the changes and ensure the project builds correctly.
  - Command: `npm run build`

## Verification Criteria
1. Tutorial balloon does not overflow screen on mobile viewports.
2. "Next/Previous" buttons are reachable even with long text.
3. Highlighting a sidebar item on mobile points to the mobile menu trigger.
4. Project builds without errors.
