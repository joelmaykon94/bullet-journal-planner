# Plan 21.1: Mostrar lista de links como badges em vez de subtarefas

## Objective
Remove the creation of subtasks from parsed task URLs. Instead, save all extracted links to the task's `link` property as a space-separated list, and update the task card renderer to parse and display them all as distinct clickable badges.

## Tasks
- [ ] **Task 1: Disable subtask creation from parsed links in useBujoItems.ts**
  - Update `handleSaveStandardInput` to not convert extracted links to subtasks. Set `subtasks` to `undefined` (or empty) if no other subtask source exists, and save all links: `link: link || (links.length > 0 ? links.join(' ') : undefined)`.
  - Update `handleSaveEditItem` to not append extracted links as subtasks, and update `link` with: `link: link !== undefined ? (link.trim() === '' ? undefined : link.trim()) : (links.length > 0 ? links.join(' ') : item.link)`.
- [ ] **Task 2: Disable subtask creation from parsed links in handleSaveRapidLog in BujoContext.tsx**
  - Update `handleSaveRapidLog` to not convert extracted links to subtasks. Set `subtasks` to `undefined` (or empty) if no other subtask source exists, and save all links: `link: links.length > 0 ? links.join(' ') : undefined`.
- [ ] **Task 3: Render multiple badges in BulletItem.tsx**
  - Update `renderTextAndBadges` to split or extract all URLs from `legacyLink` using regex instead of pushing the raw string.
- [ ] **Task 4: Verify task build compiles successfully**
  - Compile the application and ensure no type or compilation errors exist.

## Verification Criteria
1. Adding a task with multiple links (e.g. `https://google.com https://github.com`) does not create subtasks.
2. The task card displays multiple badges, one for each link (e.g., `🔗 google.com` and `🔗 github.com`).
3. The project builds cleanly using `npm run build`.
