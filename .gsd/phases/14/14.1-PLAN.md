# Plan 14.1: Ponytail Refactoring - Component & Context Cleanup

## Objective
Apply the "Ponytail" philosophy (senior laziness/efficiency) to reduce code complexity, remove redundancies, and clean up the massive context.

## Context
Phase 13 is complete. The audit revealed significant prop drilling in `BulletItem.tsx` and duplication of autocomplete logic across features. `BujoContext.tsx` is also oversized and contains many constants that should be externalized.

## Tasks

- [ ] **Task 1: Refactor `BulletItem.tsx` to use Context directly**
  - Type: auto
  - Description: Remove redundant props (`cycleStatus`, `handleDeleteItem`, `handleAISplitTask`, etc.) and consume them directly from `useBujo()`. This reduces component API surface and boilerplate in `DailyLogTab.tsx`.
  - Files: `src/features/planner/components/BulletItem.tsx`, `src/features/planner/components/DailyLogTab.tsx`

- [ ] **Task 2: Centralize `CTX_SUGGESTIONS` and Autocomplete Constants**
  - Type: auto
  - Description: Move `CTX_SUGGESTIONS` and other shared constants from components to a central `src/utils/constants.ts` or similar.
  - Files: `src/features/planner/components/DailyLogTab.tsx`, `src/features/planner/components/BulletItem.tsx`, `src/utils/constants.ts`

- [ ] **Task 3: Externalize Constants from `BujoContext.tsx`**
  - Type: auto
  - Description: Move non-reactive data (like the massive help dictionary if it was there, or default settings) out of the 2300+ line context file to reduce its size.
  - Files: `src/context/BujoContext.tsx`

## Verification Criteria
1. Project builds cleanly (`npm run build`).
2. `BulletItem` functionality (cycle, delete, edit, subtasks) remains 100% operational.
3. Code is noticeably cleaner and redundant props are gone.
4. No regression in autocomplete functionality.
