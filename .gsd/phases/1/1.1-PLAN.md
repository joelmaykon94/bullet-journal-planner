# Plan 1.1: Docker Compose PostgreSQL and Prisma Initialization

<objective>
Configure the local relational database environment using Docker Compose and establish the Prisma ORM setup inside the NestJS backend to lay down the database layer.

Purpose: Provide a reliable PostgreSQL instance and schema management tools for backend CRUD persistence.
Output: docker-compose.yml and initialized prisma directory with basic settings.
</objective>

<context>
Load for context:
- .gsd/SPEC.md
- .gsd/STACK.md
</context>

<tasks>

<task type="auto">
  <name>Create Docker Compose configuration</name>
  <files>docker-compose.yml</files>
  <action>
    Create a root-level `docker-compose.yml` specifying:
    - PostgreSQL service (image: `postgres:16-alpine`).
    - Environment variables: `POSTGRES_USER=bujo_user`, `POSTGRES_PASSWORD=bujo_pass`, `POSTGRES_DB=bujo_focus`.
    - Ports mapping: `5432:5432`.
    - Volumized storage for data persistence.
    AVOID: using generic root passwords; keep credentials explicit and clean.
  </action>
  <verify>docker compose config passes syntax verification</verify>
  <done>docker-compose.yml exists and matches PostgreSQL setup</done>
</task>

<task type="auto">
  <name>Initialize Prisma in NestJS backend</name>
  <files>apps/backend/prisma/schema.prisma</files>
  <action>
    Install Prisma dependencies in `apps/backend/` and initialize it:
    - Add `prisma` and `@prisma/client` to apps/backend/package.json.
    - Create `apps/backend/prisma/schema.prisma` with datasource "postgresql" pointing to environment DATABASE_URL.
    AVOID: running global prisma commands; run them through pnpm workspace scope `pnpm --filter backend exec prisma`.
  </action>
  <verify>apps/backend/prisma/schema.prisma exists and datasource is configured to postgresql</verify>
  <done>Prisma configurations are set up and target schemas folder exists</done>
</task>

<task type="auto">
  <name>Configure local env variables</name>
  <files>apps/backend/.env</files>
  <action>
    Create apps/backend/.env (copied from a template or generated) and set:
    - `DATABASE_URL="postgresql://bujo_user:bujo_pass@localhost:5432/bujo_focus?schema=public"`
    AVOID: exposing this connection string to the public repository. Ensure gitignore protects .env.
  </action>
  <verify>cat apps/backend/.env matches connection string</verify>
  <done>Backend .env file holds local database credentials</done>
</task>

</tasks>

<verification>
After all tasks, verify:
- [ ] docker compose config validates successfully.
- [ ] Prisma schema reads the datasource config.
</verification>

<success_criteria>
- [ ] Database configuration is complete and ready to run.
- [ ] Prisma CLI commands can target the postgres database schema.
