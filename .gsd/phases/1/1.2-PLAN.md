<objective>

Output: Extended schema.prisma with models and generated migration SQL scripts.
</objective>

<task type="auto">
  <name>Define Prisma schemas for Bullet Journal domain</name>
  <files>apps/backend/prisma/schema.prisma</files>
  <action>
    Add the following models inside `schema.prisma`:
    - `User`: id, email, passwordHash, xp, level, anxietyScore, createdAt
    - `JournalItem`: id, userId, type (TASK, EVENT, NOTE), title, categoryEmoji, targetDate, targetTime, indexTabPosition, isCompleted, isArchived, links (json or list), timeBlocked (minutes)

    Ensure proper primary key and foreign key relations (linking to User).
    AVOID: unstructured field definitions; validate relational constraints.

  <verify>pnpm --filter backend exec prisma validate succeeds</verify>
  <done>Models defined correctly with relational mapping</done>
