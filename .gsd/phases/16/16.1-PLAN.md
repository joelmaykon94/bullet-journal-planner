# Plan 16.1: Drag-and-Drop Reordering for Dreams and Achievements

## Objective
Enable drag-and-drop reordering for dreams and achievements in both the `DreamBoardTab` and the `IndexTab` (Objetivos de Hoje dashboard card). Ensure that dragging items in either view reorders the underlying dreams array, which propagates and saves to localStorage instantly, keeping both tabs perfectly synchronized.

## Tasks
- [ ] **Task 1: Implement Drag-and-Drop in DreamBoardTab**
  - Wrap the dreams rendering block in `DreamBoardTab.tsx` with a `DndContext` and `SortableContext` using `rectSortingStrategy`.
  - Use `SortableItem` and include a `DragHandle` with the `GripVertical` icon in the card header.
- [ ] **Task 2: Implement Drag-and-Drop in IndexTab**
  - Import `@dnd-kit/core`, `@dnd-kit/sortable`, and the `SortableItem` / `DragHandle` utilities in `IndexTab.tsx`.
  - Destructure `handleReorderDreams` from the `useBujo()` context.
  - Wrap the goals list in `IndexTab.tsx` inside a `DndContext` and `SortableContext` using `verticalListSortingStrategy`.
  - Convert goal item cards into `SortableItem` elements and add a `DragHandle` handle with `GripVertical` icon for dragging.
- [ ] **Task 3: Production Build & Validation**
  - Ensure the project builds cleanly without TypeScript or bundler errors.

## Verification Criteria
1. On the Dream Board tab, dragging and dropping a dream card to a new position reorders it correctly.
2. In the Central de Foco (Index Tab) "Objetivos de Hoje" card, dragging and dropping an active dream reorders it.
3. Reordering in the Dream Board is reflected in the Index Tab list, and reordering in the Index Tab list is reflected in the Dream Board.
4. Drag-and-drop functions correctly with drag handle interaction, preventing conflict with clicking buttons like "Conquistar" or "Remover".
5. The project builds cleanly using `npm run build`.
