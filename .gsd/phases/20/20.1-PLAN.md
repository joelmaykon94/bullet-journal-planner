# Plan 20.1: Mostrar os links no card da tarefa

## Objective
Ensure task links are persisted correctly and displayed as clickable domain badges on task cards. Enable extracting links automatically from content as well as manually inputting/updating them via the inline edit form.

## Tasks
- [ ] **Task 1: Persist link parameter in useBujoItems.ts**
  - Update `handleSaveStandardInput` to store `link: link || (links.length > 0 ? links[0] : undefined)` in both single and recurring tasks.
  - Update `handleSaveEditItem` to store `link: link !== undefined ? (link.trim() === '' ? undefined : link.trim()) : (links.length > 0 ? links[0] : item.link)`.
- [ ] **Task 2: Store parsed links in handleSaveRapidLog in BujoContext.tsx**
  - Update `handleSaveRapidLog` to assign `link: links.length > 0 ? links[0] : undefined`.
- [ ] **Task 3: Add link field in BulletItem.tsx inline edit form**
  - Add state `const [localLink, setLocalLink] = useState(item.link || '');` and update it on editing activation inside `useEffect`.
  - Pass `localLink` as the last parameter to `handleSaveEditItem` within `handleSaveEdit`.
  - Render an input field for "Link (Opcional)" in the editing UI below the Date/Time/Delegation section.
- [ ] **Task 4: Verify task build compiles successfully**
  - Compile the application and ensure no type or compilation errors exist.

## Verification Criteria
1. When creating a task with a link in its text (e.g. `https://google.com`), the link is extracted and shown as a clickable domain badge `🔗 google.com` on the task card.
2. In the inline edit form of a task, a manual "Link (Opcional)" text input field is displayed.
3. Editing a link manually or deleting a link in the edit form updates the badge on the task card accordingly.
4. The project builds cleanly using `npm run build`.
